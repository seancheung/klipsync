# KlipSync · 需求文档

> 上游：[想法概述](./idea.md) ｜ 下游：[产品设计](./product.md)

## 1. 背景与目标

KlipSync 是一个部署在家庭 NAS 上的 **Web 优先** 共享剪贴板。目标：

- 让 NAS 主人在自己的 Windows / macOS / iOS 设备之间**秒级**传递文本和小附件，**无需安装任何原生客户端或插件**
- 顺带让家庭成员各自拥有独立的剪贴板空间
- 通过"当前剪贴板的二维码"实现跨设备快速跳转

仅面向内网 / VPN 使用，不考虑公网暴露、不做 E2EE、不做系统剪贴板自动同步（该路径让给 ClipCascade 等原生方案）。

## 2. 用户角色

| 角色 | 描述 | 主要任务 |
| --- | --- | --- |
| **管理员（Admin）** | 部署 NAS 的主人，通常只有一个 | 管理自己的剪贴板空间；新增 / 删除普通用户；重置用户密码；查看各用户空间的非敏感使用统计；配置系统参数（附件大小上限、公开访问基址等） |
| **普通用户（User）** | 家庭成员 | 管理自己的剪贴板空间（新建 / 编辑 / 删除 / 置顶 / 清空） |

> **核心数据模型**：`用户 → 空间（Space，该用户所有剪贴板的集合） → 多个剪贴板（Clipboard）`。一个剪贴板 = 一块可编辑文本 + 0~N 个附件。没有独立的"条目 / 历史"概念，**剪贴板本身就是最小单位**。

## 3. 功能需求

### 3.1 认证

- [P0] FR-001：作为任一用户，我希望用**用户名 + 密码** 登录，以便进入自己的剪贴板空间
- [P0] FR-002：作为任一用户，我希望能登出，以便切换账号或下线
- [P1] FR-003：作为任一用户，我希望能修改自己的密码
- [P1] FR-004：作为任一用户，我希望登录状态能长期保持（长期 Cookie / refresh token），避免每次打开浏览器都要重登
- [P0] FR-005：作为新用户或密码刚被管理员重置的用户，我被**强制**在首次登录后立即修改初始密码才能进入系统，以便保证账号安全

### 3.2 剪贴板（核心）

- [P0] FR-010：作为任一用户，我希望看到自己空间的**剪贴板列表**（默认排序：置顶在前，其余按最近更新时间倒序），每项显示文本前几行 + 附件数量图标 + 最后更新时间
- [P0] FR-011：作为任一用户，我希望点击列表项**打开（激活）**该剪贴板，进入其编辑视图
- [P0] FR-012：作为任一用户，当我空间为空或主动点"新建"时，我看到一个**空白编辑区**，可以直接开始输入
- [P0] FR-013：作为任一用户，我希望在激活的剪贴板里**编辑文本**
- [P0] FR-014：作为任一用户，我希望在空白编辑区输入文本或上传附件后，系统**自动落盘为一个新剪贴板**（无需显式"保存"）
- [P0] FR-015：作为任一用户，当我在空白编辑区没有任何输入 / 上传就离开时，**不生成剪贴板**
- [P0] FR-016：作为任一用户，我希望**一键复制**当前剪贴板的文本到系统剪贴板
- [P0] FR-017：作为任一用户，我希望向激活的剪贴板**上传附件**（桌面端：粘贴截图 / 拖放文件 / 选文件；移动端：相册 / 文件选择），一个剪贴板可挂 0~N 个附件，允许任意文件类型，单文件大小受系统配置限制（默认 10MB）
- [P0] FR-018：作为任一用户，我希望**删除**某个剪贴板（连同其所有附件）
- [P1] FR-019：作为任一用户，我希望**下载**附件到本地
- [P1] FR-020：作为任一用户，我希望**删除**单个附件（不影响剪贴板本身及其他附件）
- [P1] FR-021：作为任一用户，我希望对某个剪贴板**置顶 / 取消置顶**
- [P1] FR-022：作为任一用户，我希望**清空**当前剪贴板的所有内容（文本 + 附件），但剪贴板本身保留

### 3.3 多端同步

- [P0] FR-030：作为任一用户，当我同时在多台设备上打开**同一个剪贴板**时，其内容变更（文本编辑、附件增删）在各端之间秒级同步（目标延迟 < 2s）
- [P0] FR-031：作为任一用户，我希望**剪贴板列表**的结构变更（新增、删除、置顶状态）在同账号多端之间实时同步
- [P0] FR-032：作为任一用户，当我当前激活的剪贴板在另一端被删除时，本端的编辑区显示「此剪贴板已被删除」提示并返回空白编辑区 / 列表

> 注：列表只同步结构变更，不下发"某剪贴板内容有更新"的红点。内容同步只发生在"两端同时打开同一剪贴板"时。

### 3.4 二维码快速跳转

- [P0] FR-040：作为任一用户，我希望在激活的剪贴板视图上看到**当前剪贴板链接**的二维码，扫码后能在另一设备打开同一剪贴板（仍需登录 / 授权）

> 二维码**不是分享机制**，仅用于同一用户跨设备的快捷跳转。扫码者若登录的不是该剪贴板所有者的账号，将显示"此剪贴板不存在或无权访问"。

### 3.5 管理员专属

- [P0] FR-050：作为管理员，我希望**新增**普通用户（设置用户名 + 初始密码）
- [P0] FR-051：作为管理员，我希望**删除**普通用户（连带其空间的所有剪贴板与附件）
- [P0] FR-052：作为管理员，我希望**重置**普通用户的密码（重置后该用户下次登录需按 FR-005 强制改密）
- [P1] FR-053：作为管理员，我希望查看各用户空间的**非敏感使用统计**（剪贴板数、存储占用），**不能查看剪贴板内容**
- [P1] FR-054：作为管理员，我希望在后台 UI 中**配置附件大小上限**（默认 10MB）
- [P1] FR-055：作为管理员，我希望在后台 UI 中**配置公开访问基址**（用于生成二维码中的链接；留空则自动使用当前请求的访问地址作为基址）

> FR-055 动机：自托管场景下服务常跑在 NAS 局域网 IP / 反代域名 / localhost，桌面浏览器看到的访问地址不一定是手机（或其他扫码设备）能直接打开的地址。空值表示"未设置"，此时由服务端回退到 request host 派生，保留最小部署场景零配置。

### 3.6 部署与系统

- [P0] FR-060：**Docker Compose 一键部署**
- [P0] FR-061：**数据落本地磁盘** / NAS 挂载卷，不依赖任何外部 SaaS
- [P0] FR-062：**首次部署引导** 创建管理员账号
- [P2] FR-063：支持 **PWA "添加到主屏幕"** 图标入口（不做 Web Share Target）

## 6. 范围外（Out of Scope）

- ❌ 自动监听 / 写回系统剪贴板（让给 ClipCascade 等原生客户端方案）
- ❌ Markdown 渲染、标签、分类、全文搜索
- ❌ 协作 / 评论 / 多人实时编辑 / 权限矩阵
- ❌ 公网暴露支持 / E2EE / 零信任架构
- ❌ 附件超过配置上限（默认 10MB）
- ❌ 用户自助注册（仅管理员能加人）
- ❌ 剪贴板 / 附件自动过期（完全交由用户手动管理，保持 self-hosted 最大数据自由度）
- ❌ 剪贴板命名（无标题、无标签）
- ❌ 临时访客 / 匿名分享链接（扫码者也必须是该剪贴板所有者本人）

## 附：关键用户流程

### 流程 A · 首次部署（管理员）

```
docker compose up
 ↓
首次访问 → 引导页「创建管理员账号」
 ↓
输入用户名 + 密码 → 提交
 ↓
创建成功 → 自动登录 → 进入空白编辑区
```

### 流程 B · 日常使用（电脑 → 手机传一段话）

```
设备 A（桌面）：打开 URL → 登录
 ↓
首次登录 → 空间为空 → 中间显示空白编辑区
 ↓
Ctrl+V 粘贴一段文本（或直接打字）
 ↓
内容自动落盘为新剪贴板 → 出现在列表
 ↓
设备 B（手机 · 已登录同账号）打开 URL
 ↓
列表秒级出现新剪贴板 → 点击打开
 ↓
文本可见 → 点「复制」→ 粘贴到别处使用
```

### 流程 C · 二维码快速跳转

```
设备 A（桌面）激活某剪贴板 → 视图上展示此剪贴板链接的二维码
 ↓
拿起手机 → 系统相机扫码 → 在浏览器打开链接
 ↓
未登录 → 跳登录页 → 输入凭据 → 打开该剪贴板
已登录 → 直接打开该剪贴板
 ↓
（若扫码所得剪贴板 ID 不属于当前登录账号 → 显示「此剪贴板不存在或无权访问」）
```

### 流程 D · 管理员添加家庭成员

```
管理员登录 → 进入「用户管理」页
 ↓
点击「新增用户」→ 填用户名 + 初始密码 → 确认
 ↓
管理员线下把「访问地址 + 用户名 + 初始密码」告诉家人
 ↓
家人用此凭据登录 → 被强制修改初始密码（FR-005）→ 进入系统
```

### 流程 E · 多端同步中的删除冲突

```
设备 A 和设备 B 同时打开剪贴板 X
 ↓
设备 A 在列表上删除 X
 ↓
设备 B 的列表秒级同步移除 X
 ↓
设备 B 当前正打开着 X → 编辑区显示「此剪贴板已被删除」提示
 ↓
用户点确认 → 返回空白编辑区 / 列表
```
