# KlipSync · 想法概述

## 一句话

浏览器打开即用的家庭共享剪贴板 —— 在自己的多台设备（Windows / macOS / iOS）之间秒级同步文本，按需附带图片和小文件。

## 目标用户

**主要用户**：有 NAS / 家用服务器的个人，日常同时使用 Windows、macOS、iOS 多台设备，需要频繁在这些设备之间传递一小段文本（链接、验证码、代码片段、Wi-Fi 密码等）。

**次级用户**：家庭成员偶尔共用这台 NAS 的剪贴板服务 —— 因此需要最基本的用户区分，但不上完整账号体系。

**部署与使用环境**：纯内网（家庭 Wi-Fi 或 VPN）使用，不考虑暴露到公网，也不对外提供反代教程。

## 要解决的问题

在多台设备之间快速传递一段文本是高频动作，但现有方案都有明显缺口：

- **微信 / 邮件 / 文件传输助手**：不自然、留痕、翻历史麻烦，文本和图片不统一
- **Pushbullet 等 SaaS**：自动剪贴板同步要付费，数据跑在第三方服务器
- **ClipCascade / KDE Connect 等自建方案**：每台设备都要装原生客户端，iOS 上尤其受限，偶尔借朋友设备完全用不了
- **PairDrop / PrivateBin 等 Web 方案**：只做一次性传输或加密 paste，没有可回查的持续空间

## 核心价值

按重要性排序，前两条是真正的差异化：

1. **Web 优先、零客户端**：任何带浏览器的设备打开即用；不装 App、不装插件，iOS 上也顺畅。偶尔借来的设备也能用
2. **二维码快速跳转**：电脑端激活的剪贴板视图上展示对应链接的二维码，手机系统相机扫码即可在另一设备打开同一剪贴板（仍走正常登录 / 授权，不是对外分享机制），完全免装任何客户端
3. **轻度用户区分 + 持续空间**：家庭成员各自拥有独立空间；空间内可创建多个剪贴板，每个剪贴板可编辑、置顶、删除，数据由用户自己管理（不自动过期、无条数上限）
4. **数据留在自家 NAS**：docker 一键部署，内容落在自家磁盘

## 不做什么

- **不做笔记 / 知识库**：无 Markdown 渲染、标签、分类、全文搜索
- **不做网盘 / 大文件传输**：单文件硬限制 10MB，不对标 NextCloud
- **不做协作 / 评论 / 实时编辑**：无 @、评论、权限矩阵
- **不做系统级自动剪贴板同步**：那条路必须装原生客户端，让给 ClipCascade 这类方案
- **不做公网部署支持**：仅面向纯内网 / VPN 场景，不提供外网反代教程，也不在 E2EE / 零信任架构上投入
- **不依赖 PWA Web Share Target**：iOS Safari 对该 API 支持缺失，系统分享菜单集成不作为核心路径（PWA 安装本身依然可选，但仅用于"添加到主屏幕"的便捷入口）

## 竞品分析

| 竞品 | 定位 | 核心功能 | 可借鉴 | 要避开 |
| --- | --- | --- | --- | --- |
| **ClipCascade** | 自建系统剪贴板同步 | 原生客户端自动监听系统剪贴板 + E2EE + 文本/图片/文件 + 双模式（P2S / P2P） | 文件大小限制策略 | 必须装 Windows/macOS/Linux/Android 全套客户端；iOS 受限；KlipSync 走 Web 路线就放弃"自动同步到系统剪贴板"这条承诺 |
| **PairDrop** | AirDrop 风格 Web 文件传输 | 纯浏览器、WebRTC 直传、局域网自动发现、房间码配对 | 局域网自动发现的 UX；PWA 安装（仅"添加到主屏幕"层面） | 无历史、无持久空间，仅做点对点传输，不匹配"持续剪贴板面板"的定位 |
| **PrivateBin** | 加密 pastebin | 零知识客户端 AES 加密、短链分享、自动过期 | — | 每次生成独立短链的交互不是"共享空间"，用户心智是"发出去"而非"放那儿回头取"；KlipSync 明确不做自动过期（self-hosted 交由用户手动管理数据） |
| **pcopy** | 多端 paste 工具 | Web + CLI + curl 都能用、文件 / 剪贴板 | 多入口访问思路（Web + API） | UI 陈旧、无现代移动端体验 |
| **KDE Connect** | 局域网设备桥 | 剪贴板推送 + 通知镜像 + 文件 + 远程命令 | 设备配对流程、剪贴板推送的触发 UX | 强依赖同 Wi-Fi、跨 OS 稳定性差、无 Web 访问入口；iOS 上基本不可用 |

**差异化定位**：在"Web 优先 + 持续共享空间 + 纯内网多设备自用 + 最小用户区分 + NAS 部署 + iOS 友好"这个组合上，上述竞品都没有覆盖。ClipCascade 赢在自动化但锁定原生客户端，PairDrop 赢在零安装但无持久空间，PrivateBin 赢在隐私但是短链模型，pcopy 和 KDE Connect 形态过时或依赖同网段。KlipSync 的空档就是"浏览器打开即用的家庭共享剪贴板"。
